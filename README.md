# demoapi-petstore

Automation testing framework for the PerfDog store, built with TypeScript + Playwright.
Currently focusing on backend testing due to the early stage of the project however there is placeholder code ready for the future web UI implementation.

Project key: "PD" (Petstore Demo)

A Trello board has been created to track tasks, stories and test cases. Please reach out to the workspace manager to be added.

## 1) Current scope

- API controllers in `api/` for Petstore Swagger endpoints:
  - `petController.ts`
  - `storeController.ts`
  - `userController.ts`
- API fixtures in `fixtures/controllerManager.fixture.ts`
- Typed API models in `models/api/petStore.ts`
- Reusable API test-data builders in `tests/test-data/`
- API test suites:
  - `tests/api/postPetData.spec.ts`
  - `tests/api/createOrdersForAvailablePets.spec.ts`
- UI placeholder structure remains available in `page-objects/` and `fixtures/pageManager.fixture.ts`

## 3) Clone and install

```bash
git clone https://github.com/Escobar-Alejandro/demoapi-petstore.git
cd demoapi-petstore
npm ci
npx playwright install
```

> `npx playwright install-deps` is mainly useful for Linux CI environments.

---

## 4) Environment configuration

Base URL is selected using:
- `ENVIRONMENT` (`prod`, `staging`, `dev`)
- `config/environmentUrls.json`
- `config/environmentPicker.ts`

Default behavior currently falls back to `prod` when `ENVIRONMENT` is not provided.

### Example (PowerShell)

```powershell
$env:ENVIRONMENT='prod'; npx playwright test --project=api
```

### Example (macOS/Linux)

```bash
ENVIRONMENT=prod npx playwright test --project=api
```

---

## 5) How test execution is split

Playwright projects in `playwright.config.ts`:
- `api`: runs tests tagged with `@api`
- `chromium`, `firefox`, `webkit`: run UI tests and exclude `@api`

This prevents API tests from running 3x in browser projects.

### Tagging convention

- Every API test must include `@api` in tags.
- Missing `@api` means that test may run in browser projects by mistake.

## 6) Run tests

### API tests only for now

```bash
npx playwright test --project=api
```

### API smoke only

```bash
npx playwright test --project=api --grep "@smoke"
```

### Single API test file

```bash
npx playwright test tests/api/postPetData.spec.ts --project=api
```

## 7) Reporting

```bash
npx playwright show-report
```

Artifacts:
- `playwright-report/`
- `test-results/`
- `output/` (if generated by tests)

---

## 8) CI smoke workflow

Workflow: `.github/workflows/smoke.yml`

Current behavior:
- Manual trigger: `workflow_dispatch`
- Uses `ENVIRONMENT=prod`
- Installs dependencies and Playwright browsers
- Runs:

```bash
npx playwright test --grep '@smoke|@api'
```

- Uploads:
  - `playwright-report`
  - `output` artifacts

## 9) Current project structure

```text
.github/workflows/   # CI workflows
api/                 # API controllers
config/              # environment loading/selection
fixtures/            # Playwright fixtures (API + UI)
models/api/          # API contracts/types
page-objects/        # UI page objects (placeholder for future)
tests/api/           # API specs
tests/test-data/     # reusable data builders
```

Planned features:
- Page objects
- Dynamic test data handling to validate with upcoming UI test cases
- Integration with test case manager (like Xray or Testrail).
